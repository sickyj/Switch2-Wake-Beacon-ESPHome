🎮 ESPHome Switch 2 Wake Beacon
A dedicated ESPHome project that allows you to wake up your Nintendo Switch 2 directly from Home Assistant using an ESP32.

By capturing a paired Joy-Con's Bluetooth MAC address and payload, this beacon hardware-spoofs its own Bluetooth radio to perfectly impersonate your controller, waking the console instantly without interrupting your ESP32's Wi-Fi connection to Home Assistant.

🤖 Note: The ESPHome YAML configuration and C++ hardware-spoofing logic in this repository were entirely generated by Google's Gemini AI, and based on research by alexvnesta

✨ Features
1-Click Installation: Zero C++ knowledge required. Just import the package into your ESPHome dashboard.

Hardware-Level Spoofing: Uses deep ESP-IDF APIs to overwrite the physical Bluetooth MAC address before the radio initializes, bypassing ESPHome's strict background scanner limits.

Wi-Fi Safe: Strictly targets the ESP_MAC_BT interface, ensuring your ESP32 stays perfectly connected to Home Assistant.

Flash Memory Persistence: Captures and permanently saves your Joy-Con's payload to NVS. You only have to pair it once!

🛠️ Prerequisites
An ESP32 microcontroller (ESP32-WROOM, FireBeetle32, etc.).

Home Assistant with the ESPHome add-on installed (v2024.11.0 or newer).

Your Nintendo Switch 2 and a paired Joy-Con or Pro Controller.

🚀 Installation
Thanks to ESPHome's packages feature, you do not need to copy and paste hundreds of lines of complex code or mess with C++ compiler rules.

Create a new device in your ESPHome dashboard.

Open the .yaml configuration file for the device.

Paste the following code block at the bottom of your configuration:

YAML
# Import the AI-generated Switch 2 Wake Beacon logic
packages:
  switch2_wake:
    url: https://raw.githubusercontent.com/YourUsername/YourRepo/main/switch2_master.yaml
    refresh: 0d
(Note: Ensure your ESP32 is set to use the esp-idf framework in your base config, as the standard Arduino framework does not support the necessary Bluetooth stack for this project).

Click Install and flash the firmware to your ESP32!

🎮 How to Use
Once installed, two new entities will appear in your Home Assistant dashboard: a Capture Mode switch and a Wake Switch 2 button.

Step 1: The Capture Phase (Do this once)
Your ESP32 needs to learn what your Joy-Con looks like so it can mimic it.

In Home Assistant, turn ON the Capture Mode switch.

Wake up your Switch 2 controller by pressing any button.

Bring the controller close to the ESP32.

The ESP32 will automatically intercept the controller's BLE advertisement, save its MAC address and payload to flash memory, and automatically reboot to apply the hardware MAC spoofing.

(You can verify this worked by looking for the "Saved Nintendo Payload" text sensor in Home Assistant!)

Step 2: Wake the Console
Once the ESP32 reboots, the setup is complete forever.

Simply press the Wake Switch 2 button in Home Assistant.

The ESP32 will broadcast the exact 31-byte raw BLE advertisement with the 0x81 wake flag, and your console will turn on!

🧠 Under the Hood (For the nerds)
Why does this project use an external custom_mac.h helper file?

The Nintendo Switch strictly validates the Public MAC address of the device attempting to wake it. Dynamically forcing an ESP32 to change its BLE MAC address while ESPHome's background tracker is running causes severe radio crashes (can't set randaddress 1).

To solve this, this project utilizes esp_iface_mac_addr_set(custom_mac, ESP_MAC_BT) during an on_boot hook at priority: 250 (after NVS loads, but before the BLE radio turns on). Because C++ strictly requires #include <esp_mac.h> to be placed in the global scope rather than inside an ESPHome lambda, the master YAML dynamically downloads a remote header file during compilation to satisfy the C++ compiler while keeping the user's YAML incredibly clean.
